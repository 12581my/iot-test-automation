# 缺陷报告示例

## BUG_001 新增设备时允许重复设备编号

| 字段 | 内容 |
|---|---|
| 缺陷编号 | BUG_001 |
| 缺陷标题 | 新增设备时允许重复设备编号 |
| 所属模块 | 设备管理 |
| 严重程度 | 严重 |
| 优先级 | P0 |
| 前置条件 | 系统中已存在设备编号 DEV001 |
| 测试环境 | Chrome / Windows |
| 复现步骤 | 1. 登录管理员账号 2. 进入设备管理页 3. 点击新增设备 4. 输入设备编号 DEV001 5. 点击保存 |
| 实际结果 | 系统提示新增成功，设备列表中出现两个 DEV001 设备 |
| 预期结果 | 系统应提示设备编号已存在，不允许保存 |
| 附件 | 暂无 |
| 状态 | 待修复 |

## BUG_002 未登录用户可以直接访问设备列表页面

| 字段 | 内容 |
|---|---|
| 缺陷编号 | BUG_002 |
| 缺陷标题 | 未登录用户可以直接访问设备列表页面 |
| 所属模块 | 权限管理 |
| 严重程度 | 严重 |
| 优先级 | P0 |
| 前置条件 | 用户未登录 |
| 测试环境 | Chrome / Windows |
| 复现步骤 | 1. 清除浏览器登录状态 2. 在地址栏直接输入设备列表页面 URL |
| 实际结果 | 页面正常展示设备列表 |
| 预期结果 | 系统应跳转到登录页，或提示用户未登录 |
| 附件 | 暂无 |
| 状态 | 待修复 |

| 字段 | 内容 |
|---|---|
| 缺陷编号 | BUG_003 |
| 缺陷标题 | 设备名称为空时仍可保存成功 |
| 所属模块 | 设备管理 |
| 严重程度 | 一般 |
| 优先级 | P1 |
| 前置条件 | 管理员已登录 |
| 测试环境 | Chrome / Windows |
| 复现步骤 | 1. 进入设备管理页 2. 点击新增设备 3. 设备名称留空 4. 输入设备编号 5. 点击保存 |
| 实际结果 | 系统保存成功 |
| 预期结果 | 系统应提示设备名称不能为空 |

| 状态 | 待修复 |
