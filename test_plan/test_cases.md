# IoT 设备管理平台测试计划

## 1. 项目背景

本项目模拟一个物联网设备管理平台，主要用于管理设备信息、查看设备在线状态、接收设备上报数据、处理设备告警记录，并支持不同角色用户进行权限管理。

本测试项目用于练习软件测试完整流程，包括功能测试、接口测试、UI 自动化测试、性能测试和持续集成。

## 2. 测试目标

- 验证用户登录、设备管理、设备状态查询、设备数据上报、告警记录等核心功能是否符合预期。
- 验证接口返回数据是否正确，接口异常场景是否有合理提示。
- 验证页面主要操作流程是否可正常完成。
- 验证高频接口在一定并发下的响应时间和稳定性。
- 通过自动化测试提高回归测试效率。

## 3. 测试范围

### 3.1 用户模块

- 用户登录
- 用户退出
- 登录失败提示
- Token 鉴权

### 3.2 设备管理模块

- 新增设备
- 查询设备列表
- 编辑设备信息
- 删除设备
- 按设备名称、设备编号、状态筛选设备

### 3.3 设备状态模块

- 查看设备在线状态
- 查看设备离线状态
- 查看设备最后上报时间

### 3.4 数据上报模块

- 设备上报温度、湿度、电量等数据
- 校验上报数据格式
- 查询设备历史数据

### 3.5 告警模块

- 查询告警记录
- 按告警级别筛选
- 处理告警
- 查看告警详情

### 3.6 权限模块

- 管理员权限
- 普通用户权限
- 未登录访问限制

## 4. 不测试范围

- 真实硬件设备接入
- 真实短信、邮件、微信告警推送
- 大规模生产环境性能测试
- 第三方云平台真实对接

## 5. 测试类型

- 功能测试
- 接口测试
- UI 自动化测试
- 性能测试
- 回归测试
- 兼容性测试

## 6. 测试环境

| 类型 | 内容 |
|---|---|
| 操作系统 | Windows / Linux |
| 浏览器 | Chrome |
| 数据库 | MySQL |
| 接口测试工具 | Apifox / Postman |
| 自动化框架 | Python + Pytest + Requests |
| UI 自动化工具 | Playwright |
| 性能测试工具 | JMeter |
| 代码管理 | Git + GitHub |
| 持续集成 | GitHub Actions |

## 7. 测试准入标准

- 被测系统核心功能可以正常访问。
- 登录、设备管理、告警查询等主要接口可用。
- 测试环境部署完成。
- 测试账号和测试数据准备完成。

## 8. 测试通过标准

- 核心功能测试用例通过率达到 95% 以上。
- 阻塞级和严重级缺陷全部修复。
- 接口自动化用例可以稳定执行。
- 主要接口无明显超时或异常错误。
- 测试报告、缺陷报告和自动化测试报告完整输出。

## 9. 风险说明

- 当前项目为学习项目，部分业务场景使用模拟数据。
- 暂不接入真实物联网硬件设备。
- 性能测试结果仅用于学习和基础分析，不代表真实生产环境表现。
